2. Create `.github/workflows/ci.yml`:

   ```yaml
   name: CI
   on:
     push:
       branches: [main]
     pull_request:
       branches: [main]

   jobs:
     lint:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v6
         - uses: actions/setup-node@v6
           with:
             node-version: "22"
         - run: npm ci
         - run: npm run lint
         - run: npm run format:check

     typecheck:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v6
         - uses: actions/setup-node@v6
           with:
             node-version: "22"
         - run: npm ci
         - run: npm run typecheck

     test:
       strategy:
         matrix:
           os: [ubuntu-latest, windows-latest, macos-latest]
       runs-on: ${{ matrix.os }}
       steps:
         - uses: actions/checkout@v6
         - uses: actions/setup-node@v6
           with:
             node-version: "22"
         - run: npm ci
         - run: npm run test:coverage
   ```

   Design decisions per user decisions and research:
   - Lint + typecheck on Ubuntu only (platform-independent)
   - Tests on all 3 OS via matrix (the actual cross-platform requirement)
   - Node 22 (active LTS through Apr 2027, ESLint 10 requires >= 20.19.0)
   - `npm ci` for reproducible installs
   - `test:coverage` runs tests with coverage threshold enforcement
   - CI fails on lint/format/typecheck/test violations -- no auto-fix commits

3. Create `.github/workflows/publish.yml`:

   ```yaml
   name: Publish
   on:
     push:
       tags: ["v*"]

   jobs:
     publish:
       runs-on: ubuntu-latest
       permissions:
         contents: write
         id-token: write
       steps:
         - uses: actions/checkout@v6
         - uses: actions/setup-node@v6
           with:
             node-version: "22"
             registry-url: "https://registry.npmjs.org"
         - run: npm ci

         - name: Validate version match
           run: |
             TAG_VERSION="${GITHUB_REF#refs/tags/v}"
             PKG_VERSION=$(node -p "require('./package.json').version")
             if [ "$TAG_VERSION" != "$PKG_VERSION" ]; then
               echo "::error::Tag version ($TAG_VERSION) does not match package.json version ($PKG_VERSION)"
               exit 1
             fi

         - run: npm run test

         - run: npm publish --provenance --access public
           env:
             NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

         - uses: softprops/action-gh-release@v2
           with:
             generate_release_notes: true
   ```

   Design decisions per user decisions and research:
   - Triggered only by `v*` tags (manual tag push by maintainer)
   - `permissions: contents: write` required for GitHub Release creation
   - `permissions: id-token: write` enables npm provenance (supply chain security)
   - `registry-url` in setup-node creates `.npmrc` with `NODE_AUTH_TOKEN` placeholder
   - Version validation: tag version must match package.json version (shell comparison, no third-party action)
   - Runs tests before publish as a safety net
   - `--provenance` flag for npm attestation
   - `--access public` required for first publish of unscoped package
   - `softprops/action-gh-release@v2` with `generate_release_notes: true` auto-creates GitHub Release with changelog
   - No build step -- publishes TypeScript source as-is (pi runtime handles TS natively)
   - Auth via `NPM_TOKEN` GitHub Actions secret (npm automation token, configured by user in repo settings)
