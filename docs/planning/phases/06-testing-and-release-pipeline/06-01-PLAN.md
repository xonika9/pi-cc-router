1. Run `npm run test:coverage` and review the branch coverage report to identify the actual uncovered lines/branches. Cross-reference with the RESEARCH.md gap analysis below.

2. Write new tests for the following meaningful gaps identified in RESEARCH.md (Section: "Potential Test Gaps to Investigate"). Focus on branch coverage -- these are where bugs hide:

   **provider.ts:**
   - Abort signal paths: test behavior when AbortSignal is already aborted before stream starts, and when abort fires mid-stream
   - hasPendingCustomToolResult edge cases: test the branch where custom tool results are pending at message_stop (affects break-early logic)
   - effectiveReason override logic: test that stopReason is correctly overridden when only user MCP tools were called

   **process-manager.ts:**
   - validateCliPresence error path: mock execSync to throw and verify the error message surfaces correctly
   - validateCliAuth error path: mock execSync to return non-authenticated output and verify the error message

   **prompt-builder.ts:**
   - resolveAgentsMdPath walk-up resolution: test the directory walk-up behavior when AGENTS.md exists in a parent directory but not the immediate cwd
   - sanitizeAgentsContent regex edge cases: test with content containing patterns that could trip the regex (empty content, content with only whitespace, content with special characters)

   **event-bridge.ts:**
   - Unknown event types: test that unrecognized event type strings are handled gracefully (no crash, logged appropriately)
   - Delta without matching block: test that a delta event arriving before its corresponding block_start is handled without error

3. Do NOT add tests for:
   - Coverage number padding (trivial getters/constructors)
   - Process exit handlers (untestable without real subprocess)
   - console.warn/console.error output verification (already covered implicitly)
   - Any path where writing the test would be more complex than the code under test
