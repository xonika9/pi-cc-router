<objective>
Implement configurable thinking effort mapping (CONF-02) that maps pi's ThinkingLevel to Claude CLI --effort flags, with elevated mapping for Opus models.

**Note:** CONTEXT.md specified `--thinking-budget <tokens>` and `mapThinkingTokens()`, but RESEARCH.md discovered `--thinking-budget` does not exist on the Claude CLI (v2.1.76). The actual flag is `--effort <level>`. This plan adapts accordingly: `mapThinkingEffort()` returns `CliEffortLevel` instead of token counts. The Opus elevation concept is preserved via effort-level shifting.

Purpose: Pi users selecting a reasoning level (minimal/low/medium/high/xhigh) need that mapped to the Claude CLI's --effort flag. Opus models get shifted-up effort (e.g., pi "medium" becomes CLI "high") to leverage their superior reasoning capability.

Output: New thinking-config.ts module, modified process-manager.ts and provider.ts, updated index.ts wiring, and comprehensive tests.
</objective>
